# Preprocessing Summary

This file summarizes the preprocessing used for the reproducibility package and manuscript benchmarks.

## Synthetic datasets

The seven synthetic datasets are generated by the included project code with random seed 42. Continuous columns are generated from Gaussian cluster-specific centers and categorical columns are generated from cluster-dependent categorical templates with controlled noise. The `true_label` column is retained only for external evaluation metrics such as ARI and NMI. It is not used by K-Prototypes fitting, gamma search, or internal-objective optimization.

## Real datasets

The four primary real datasets are loaded from the UCI Machine Learning Repository through the `ucimlrepo` interface in `experiments/run_real_benchmark.py`.

### Missing values

Categorical missing values and placeholder values such as `?` are replaced by the string `missing`. Numeric missing values are imputed with the median of the corresponding feature.

### Categorical features

For Heart Disease and Bank Marketing, categorical columns are specified manually to avoid ambiguous type inference. For Credit Approval and Adult, categorical columns are detected from object, category, or Boolean columns. If no categorical columns are detected, low-cardinality columns with at most 10 unique values are treated as categorical by the loader.

### Continuous features

Continuous columns are coerced to numeric values. Missing numeric values are replaced with the feature median. Before K-Prototypes fitting, continuous columns are standardized with `StandardScaler` and categorical columns are kept as strings.

### Labels

Labels are used only for external evaluation and diagnostics. Gamma search optimizes the internal K-Prototypes objective and does not use labels. Heart Disease labels are binarized as 0 versus greater than 0 disease status. Adult labels are cleaned by removing trailing periods from test-set label strings. Other labels are label-encoded as strings for metric computation.

### Sampling

The real-data benchmark script supports a `--max-samples` option. The manuscript benchmark used the documented settings in the result files. If a dataset exceeds the specified maximum sample size, the loader samples rows without replacement using the configured random seed.
